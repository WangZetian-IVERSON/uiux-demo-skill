# Style Directions

## Direction Fallback

When the user says "make it look good", "make a portfolio", or gives incomplete style guidance, offer 2-3 directions before producing the full deck. Each direction must differ in more than color.

For each direction, provide:

- Name.
- Best use case.
- Visual signature.
- Page rhythm.
- Risks.
- 2-3 sample page concepts.

## Direction A: Dark-Tech Portfolio

Best for:
Enterprise, data dashboards, command systems, AI products, industrial platforms, and designers who want a high-impact visual portfolio.

Visual signature:
Black/blue cinematic canvas, glass panels, neon-blue accents, device frames, metric cards, 3D perspective, grid floors.

Risks:
Can become generic cyber style if logic is weak or glow is overused.

## Direction B: Clean Product Case Study

Best for:
Hiring portfolios, SaaS, B2B tools, redesign case studies, and interview decks.

Visual signature:
Light or neutral background, strong grid, restrained colors, large UI evidence, clear before/after reasoning, fewer effects.

Risks:
Can feel too plain if no strong hierarchy or project insight is visible.

## Direction C: Editorial Design Story

Best for:
Consumer apps, lifestyle products, creator portfolios, visual identity projects, and experience design stories.

Visual signature:
Large typography, magazine-like pacing, full-bleed imagery, expressive section covers, case-study copy with a stronger narrative voice.

Risks:
Can become too brand/editorial and under-explain interaction decisions.

## Direction D: Research-Heavy UX Deck

Best for:
UX research, service design, complex workflows, early-stage product discovery, stakeholder alignment.

Visual signature:
Journey maps, matrices, quotes, synthesized findings, information architecture, and structured insight pages.

Risks:
Can become text-heavy. Must alternate research proof with clear interface implications.

## Direction E: Data Command Center

Best for:
Large-screen visualization, smart city, energy, logistics, security, operations, IoT, and monitoring systems.

Visual signature:
Map layers, dashboard panels, alert states, dark spatial environment, metrics, charts, data hierarchy, operational scenarios.

Risks:
Can turn into decorative dashboard wallpaper. Must state what each data layer helps decide.

## Direction F: Warm Commercial Portfolio

Best for:
Ecommerce apps, local services, retail, marketplace products, merchant tools, operations visuals, and designers who want friendly commercial energy.

Visual signature:
Warm orange/yellow gradients, cream or pale mint surfaces, playful 3D mascot/object, large bold Chinese titles, dense app screenshot collages, soft icon tiles, and business-goal pages.

Page rhythm:
Cover -> ecommerce project background -> screenshot overview -> optimization/value page -> component/icon library -> campaign visuals -> B-side/SaaS expansion -> thanks.

Risks:
Can feel too cute or promotional if product logic is weak. Balance playfulness with explicit efficiency, consistency, and conversion arguments.

## Direction G: B-Side SaaS Review

Best for:
Official website, admin console, merchant backend, SaaS solution pages, and enterprise operations products.

Visual signature:
Light blue/white interface evidence, cobalt accents, desktop/laptop mockups, component library boards, KPI cards, structured callouts, and clean product screenshots.

Page rhythm:
Project background -> business goals -> website/admin information architecture -> core scenario -> component library -> efficiency/outcome -> reflection.

Risks:
Can become sterile if it only shows clean screens. Add business goals, scenario storytelling, and before/after clarity.
