# Public Corpus Notes

## What Public Sources Add

Local screenshots provide strong style DNA. Public sources add broader coverage:

- Common UX case-study structures.
- Interview and hiring portfolio expectations.
- Template page inventories.
- Current portfolio presentation trends.
- Examples outside the dark-tech aesthetic.

## Sources Checked

Figma case-study templates:
Useful for common case-study inventory: cover, project overview, wireframes, journey insights, outcomes, and editable case-study structures.

Behance UI/UX portfolio and presentation searches:
Useful for seeing current visual packaging patterns, high-polish project thumbnails, complete portfolio presentation framing, and the scale of popular portfolio/case-study examples.

UI4Free Figma Behance Presentation Template:
Useful because it explicitly lists case-study page categories such as cover, introduction, problem vs solution, design process, design strategy, and user research.

Huashu Design:
Useful for skill architecture rather than UI/UX portfolio content. Borrow the process ideas: fact verification, core asset protocol, early assumptions, placeholders over weak implementation, variations, anti-slop checks, and post-output critique.

Huashu image logic:
Do not treat AI image generation as the main design method. Huashu uses a hierarchy: real assets first, official/public assets second, AI generation as fallback for missing support visuals. In its docs, `nano-banana-pro` is mentioned as an AI image fallback, especially when a real product reference exists and a scene variant is needed. For this skill, keep the tool generic: prefer `image2` if available, otherwise use any available image generation tool, otherwise output prompts only.

Authoring corpus note:
The original local UI/UX screenshot folders were used to distill this skill's rules. They should not be treated as runtime dependencies or reusable asset libraries. Use the distilled rules in this skill, then generate or collect fresh project-specific assets for each new theme.

## What Not To Copy

- Do not copy specific portfolio layouts one-for-one.
- Do not treat Behance popularity as a quality guarantee.
- Do not overfit to dark cinematic style when the project needs clarity or research credibility.
- Do not use public examples to fabricate project facts.
